Vue-自定义指令的使用

发表于 2019-08-18 | 分类于 框架 | | 阅读次数:
字数统计: 799 | 阅读时长 ≈ 2

自定义指令是用来操作DOM的。尽管Vue推崇数据驱动视图的理念，但并非所有情况都适合数据驱动。自定义指令就是一种有效的补充和扩展，不仅可用于定义任何的DOM操作，并且是可复用的。

先了解一下,在 vue 中,有很多内置的指令.

  • v-for 用于遍历

  • v-if & v-show 用于隐藏和显示元素(区别在于后者是修改 display:block|none,前者是不创建把元素从dom中删除或者创建.

  • v-bind: 属性绑定,把数据绑定在HTML元素的属性上.

  • v-html & v-text 把数据绑定在HTML元素的属性上,作用同 innerHTML & innerText

  • v-on: 绑定HTML元素事件

  • v-if & v-else-if & v-else 条件渲染

  • v-model 绑定表单元素,实现双向绑定.

这里我们就讲这些常用的就行了

自定义指令和过滤器一样都可以定义全局指令和菊部私有指令，我们用一个小demo来演示

自定义全局指令

场景：我们需要一个指令,写在某个HTML表单元素上,然后让它在被加载到DOM中时,自动获取焦点.

  • 使用 Vue.directive() 定义全局的指令 v-focus

  • 其中：参数1：指令的名称，注意，在定义的时候，指令的名称前面，不需要加 v- 前缀，但是：在调用的时候，必须在指令名称前加上v-前缀来进行调用

  • 参数2：是一个对象，这个对象身上，有一些指令相关的函数（ps：钩子函数）,这些函数可以在特定的阶段，执行相关的操作

  • 我们再自定义一个设置字体颜色的指令

  • bindling参数中的value属性容易与expression属性混淆，建议看官方文档说明。

  • 注：样式只要通过指令绑定给了元素，不管这个元素有没有被插入到页面中去，这个元素肯定有了一个内联样式 将来元素肯定会显示到页面中，这时候，浏览器的渲染引擎必然会解析样式，应用给这个元素。

  • 最后调用指令就行啦

总结：

  • 使用 Vue.directive() 来新建一个全局指令,(指令使用在HTML元素属性上的)

  • Vue.directive(‘focus’) 第一个参数focus是指令名,指令名在声明的时候,不需要加 v-

  • 在使用指令的HTML元素上,我们需要加上 v-.

  • Vue.directive(‘focus’,{}) 第二个参数是一个对象,对象内部有个 inserted() 的函数,函数有 el 这个参数.

    • el 这个参数表示了绑定这个指令的 DOM元素,在这里就是后面那个有 placeholder 的 input

    • el 就等价于 document.getElementById(‘el.id’)….可以利用 $(el) 无缝连接 jQuery
